# Compliance Audit Dashboard

Skill-building project demonstrating AI-assisted SQL for structured data auditing, validation, and compliance-style rule checking across relational datasets.

Implements a SQL-based framework for loading structured records into a database, executing validation queries, and producing audit-style outputs for data quality review.

## 🔍 What It Does

- Store and organize structured case data using a SQL schema
- Run SQL audit queries to flag missing or invalid records
- Support MySQL or PostgreSQL database environments
- Generate query-based outputs for data quality review workflows

## 🧰 Tools Used

- SQL (MySQL or PostgreSQL, AI-assisted)
- Database schema design
- SQL queries for data validation and reporting

## 📁 File Overview

- `schema.sql` — Defines database structure for structured case records
- `insert_data.sql` — Inserts sample dataset into relational tables
- `queries.sql` — SQL audit queries for validation and reporting outputs

## 🧪 How to Use It

1. Create a MySQL or PostgreSQL database
2. Run `schema.sql` to create required tables
3. Run `insert_data.sql` to populate sample records
4. Execute `queries.sql` to run audit and validation checks

## ✅ Use Case

Designed for structured data environments requiring SQL-based validation and audit-style reporting of high-volume transactional records prior to review or downstream processing.
